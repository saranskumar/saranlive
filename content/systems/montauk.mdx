## Problem

Building a technically impressive creative project that demonstrates full-stack web capabilities, UI/UX depth, and audio engineering — while telling an immersive story.

## Architecture

A Next.js 14 app structured as a fictional 1980s containment system with three main modules:

1. **Reality Monitor** - Dual-mode UI (Standard / Upside Down) with CRT emulation, chromatic aberration, and particle effects.
2. **Incident Tracker** - Live feed of threats classified by severity and dimensional origin, with a full status workflow.
3. **CMD Terminal** - Functional in-browser terminal with custom system commands (`/HELP`, `/INCIDENT`, `/STAT`, `/DOOM`).

## Implementation

- **Visual Layer**: Tailwind CSS extended with custom CRT scan-line animations and chromatic aberration filters.
- **Motion**: Framer Motion handles the boot sequence, reality-shift transitions, and particle anomaly animations.
- **Audio**: Web Audio API synthesizes a reactive soundscape — each dimension has its own audio profile (computer hums vs. organic drones), intensifying with threat level.
- **State**: LocalStorage persists session incidents and operator settings across reloads.
- **Signal Calibrator**: Interactive sliders with real-time waveform visualization and audio frequency feedback.

## Outcome

A fully deployable, immersive creative web experience demonstrating:

- Complex state management and UI orchestration
- Browser-native audio synthesis without external libraries
- Responsive, performance-optimised animation at 60fps
- Accessible keyboard-driven operator controls

## Lessons Learned

- **Web Audio API** opens powerful real-time synthesis possibilities directly in the browser — no dependency needed.
- **CRT aesthetics with CSS** can be achieved cleanly with a small set of layered filter and animation utilities.
- **Framer Motion + Next.js** works especially well for phased boot sequences with precise timing.
